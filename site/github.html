GitHub

A home for your code — and a place to collaborate with others.

What is GitHub?

GitHub is a website where you store your code and share it with others. It's built on top of Git (the version control tool), but it adds a friendly web interface, collaboration features, and a whole ecosystem of tools on top.

When you push your code to GitHub, it lives in a "repository" — think of it as a project folder that's hosted online. Other people can see your code, suggest changes, or even copy it to build on top of it (that's called "forking").

GitHub is also where you'll access tools like Copilot, Codespaces, and GitHub Pages — all of which are covered in this tutorial. It's the central hub for everything you're building here.

Think of it like...

If Git is like saving drafts of a document on your computer, then GitHub is like Google Drive for code. It stores your files in the cloud so you can access them from anywhere, share them with others, and collaborate in real time. Just like Google Drive lets multiple people comment on a document, GitHub lets multiple developers work on the same project without stepping on each other's toes.

Why does it matter?

GitHub is where your project lives online. It's your backup, your portfolio, and your collaboration space all in one. When you push code to GitHub, it's safely stored in the cloud — and from there, you can deploy it as a website, invite others to contribute, or simply keep a record of your progress.

For this tutorial, GitHub is the foundation. Your fork, your Codespace, your published site — they all run through GitHub. Getting comfortable with it now will pay off in everything you build going forward.
