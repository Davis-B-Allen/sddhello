Get Started with AI-Assisted Development

A practical, hands-on guide to building software with an AI at your side.

What is this?

This site is a practical, hands-on guide designed to help you — someone who may have little or no experience with software development — get set up with modern tools for building software alongside an AI assistant.

The idea is ambitious but achievable: you're going to learn how to build real applications, and you won't be doing it alone. You'll have GitHub Copilot — an AI-powered coding assistant — working right beside you, ready to write code, explain concepts, and guide you through every step.

This guide follows a "learn by making" philosophy. Rather than reading a textbook cover to cover before writing a single line of code, you'll jump in, get your hands dirty, and learn as you go. When you hit something you don't understand, you'll have your AI assistant right there to explain it. The best way to learn is by doing — and with AI as your partner, you can start doing interesting things right away.

Ready? Let's get you set up.

Key Concepts

Before you dive into the setup steps, here's a quick introduction to the tools and ideas you'll be working with. Don't worry about memorizing anything — you'll learn more about each of these as you use them.

GitHub Codespaces

A Codespace is a complete development environment that runs in your web browser. Instead of installing complicated software on your computer, you click a button and get a fully configured workspace in the cloud — ready to code in seconds. Think of it as a virtual computer that's already set up with everything you need for this project.

Learn more about GitHub Codespaces →

GitHub Copilot & AI Agents

GitHub Copilot is an AI assistant that lives inside your development environment. You can talk to it in plain English — ask it to write code, explain how something works, fix a bug, or help you plan a feature. When you use Copilot in agent mode, it can take on bigger tasks: reading your project files, running commands, and making changes across multiple files on your behalf.

Learn more about GitHub Copilot →

Spec-Driven Development & spec-kit

Spec-Driven Development (SDD) is a structured way to build software. Instead of jumping straight into code, you start by writing a clear description of what you want to build — called a specification. Then you plan, break the work into tasks, and implement step-by-step. spec-kit is a toolkit from GitHub that helps you follow this process with AI. You'll use simple slash commands like /speckit.specify and /speckit.plan to guide your AI assistant through each stage.

Learn more about spec-kit →

Getting Started

Follow these steps to go from zero to a working development environment — and a live website — right in your browser. Each step explains what you're doing and why, so you can follow along confidently even if this is all new to you.

  1. Create a GitHub account

    GitHub is a platform where developers store and collaborate on code. You'll need a free account to access the project and use the tools described on this page. If you already have an account, skip ahead to step 2.

    Sign up for GitHub →

  2. Sign up for the GitHub Copilot Pro free trial

    Copilot Pro gives you access to the AI assistant that will help you write code and learn as you go. GitHub offers a free 30-day trial, so you can get started without any cost. You'll need your GitHub account from step 1.

    Start your free Copilot Pro trial →

  3. Fork the repository

    "Forking" means creating your own personal copy of the project on GitHub. This gives you a safe space to experiment and make changes without affecting the original. Visit the project's repository page on GitHub and click the Fork button near the top right of the page. Follow the prompts to create your fork.

    Go to the repository on GitHub →

  4. Open your fork in GitHub Codespaces

    Now it's time to open your development environment. Go to your fork of the repository on GitHub (not the original). Click the green <> Code button, select the Codespaces tab, and click Create codespace on main. After a minute or two, a fully configured editor will open in your browser — you're ready to start building!

  5. Preview the site locally

    Let's make sure the site works before you change anything. Open the terminal in VS Code (Terminal → New Terminal, or press Ctrl+`) and run: 

    npx serve site -p 8080

    In a Codespace, a notification will pop up saying "Your application running on port 8080 is available." Click Open in Browser to see the site. You can also open the Ports panel (in the bottom bar) and click the globe icon next to port 8080.

    Note: If you're developing locally (not in a Codespace), open http://localhost:8080 directly in your browser after running the serve command.

  6. Publish your site with GitHub Pages

    Your fork already includes a workflow file that can publish the site to the web automatically. You just need to enable it:

    1. In your fork on GitHub, go to the Actions tab. If prompted, click "I understand my workflows, go ahead and enable them".
    2. Go to Settings → Pages (in the left sidebar under "Code and automation").
    3. Under Build and deployment, set Source to GitHub Actions.
    4. Push a commit to main (or click "Run workflow" on the Actions tab) to trigger deployment.

    After a few minutes, your site will be live at https://<your-username>.github.io/<your-repo>/.

A Tip: Ask Mode

If you run into something you don't understand — a term, a concept, an error message, anything at all — you can switch your AI agent to "ask" mode at any time. Just type your question in plain English and it will explain things for you. There are no silly questions — that's exactly what your AI assistant is there for.
