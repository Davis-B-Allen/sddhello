Git & Version Control

Track changes to your code like saving drafts of a document.

What is Git?

Git is a tool that keeps track of every change you make to your files. When you're writing code (or any text, really), Git remembers what the files looked like at each point in time. If you make a mistake, you can go back to an earlier version — no panic required.

You don't need to understand everything about Git right away. At its core, it does three things for you: it saves snapshots of your work (called "commits"), it lets you see what changed between snapshots, and it lets you undo things if something goes wrong.

Git runs on your computer (or in your Codespace) and works behind the scenes. Most of the time, your AI assistant or your editor will handle the Git commands for you — but it helps to know what's happening under the hood.

Think of it like...

Imagine you're writing a school essay. Every time you finish a paragraph, you hit "Save As" and give the file a new name: essay-v1.docx, essay-v2.docx, essay-final.docx, essay-final-FINAL.docx. Sound familiar? Git does the same thing — but way more elegantly. Instead of cluttering your folder with copies, it quietly tracks every version in the background, and you can jump back to any one at any time. It's like having an unlimited undo history for your entire project.

Why does it matter?

Without version control, one wrong edit could break everything — and you'd have no easy way to get back to a working state. Git gives you a safety net. You can experiment freely, knowing that your previous work is always safe.

As you build your site and add new features, Git will keep a clean history of every change you've made. That history isn't just for backup — it also helps you (and your AI assistant) understand why something was changed and when.
