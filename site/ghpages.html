GitHub Pages

Publish a website for free, straight from your code repository.

What is GitHub Pages?

GitHub Pages is a free service from GitHub that turns the files in your repository into a live website. You don't need to rent a server, configure hosting, or pay for a domain — GitHub handles all of that for you. Just push your HTML, CSS, and JavaScript files, and your site goes live.

Your site gets a URL like https://your-username.github.io/your-repo/. Anyone with that link can visit it. Every time you push new code, the site updates automatically — usually within a minute or two.

The site you're reading right now is hosted on GitHub Pages! It started as a few files in a repository, and GitHub Pages made it available to the world with just a few clicks.

Think of it like...

Imagine you wrote a newsletter and want to share it with the world. Normally, you'd need to find a printing press, a distribution network, and a way to get it into people's hands. GitHub Pages is like one-click publishing: you write your content, click a button, and it's instantly available to anyone on the internet. No printing press required.

Why does it matter?

Publishing a website used to require understanding servers, domains, and hosting providers. GitHub Pages makes it free and simple. It's perfect for personal sites, project documentation, portfolios, and learning projects like this one.

Because it's integrated with Git and GitHub, your site updates automatically every time you push changes. That means you can edit your code, push it, and see the results live within minutes. It's one of the fastest ways to go from "code on your screen" to "website on the internet."
