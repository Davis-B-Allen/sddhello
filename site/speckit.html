Spec-Driven Development & spec-kit

A structured way to plan before you build — like a blueprint for software.

What is Spec-Driven Development?

Spec-Driven Development (SDD) is an approach where you write a clear description of what you want to build before you write any code. That description is called a "spec" — short for specification. It answers questions like: What should this feature do? Who is it for? How should it behave?

spec-kit is a toolkit that makes SDD easy to follow, especially when you're working with an AI assistant. It provides a set of slash commands — like /speckit.specify, /speckit.plan, and /speckit.tasks — that guide you through the process of turning an idea into a finished feature, one step at a time.

You don't need to be a project manager to use it. Think of it as a conversation framework: you tell the AI what you want, and spec-kit helps organize that conversation into something actionable.

Think of it like...

Imagine you want to build a house. You wouldn't just show up with a hammer and start nailing boards together — you'd start with an architectural blueprint. The blueprint tells you where the walls go, how big the rooms are, and where the plumbing runs. Spec-Driven Development is the same idea, but for software. Your spec is the blueprint, and spec-kit is the architect's drafting table that helps you draw it.

Why does it matter?

Without a plan, it's easy to get lost in the details or build something that doesn't quite match what you had in mind. A spec keeps you on track. It gives your AI assistant clear instructions, reduces back-and-forth, and makes it easier to break big ideas into small, manageable tasks.

This very page was built using spec-kit! The feature started as an idea ("add concept pages"), went through the specify → plan → tasks → implement workflow, and ended up as the page you're reading now. That's SDD in action.
